## What shipped (all merged to `main` unless noted)

### The weekend outage — root cause + structural class-fix
The bot drafted nothing all weekend (Fri 06-19 → Mon 06-23). **Root cause: a flaky
time-bomb unit test gated the production bot off.** `tests/test_scheduler.py::test_breaker_records_skipped_not_failed`
seeded source-health failures at hardcoded `2026-06-12` timestamps and asserted
`failed == 3`; the breaker appends a *now*-stamped `skipped` row and the 7-day
source-health window is anchored to the latest run, so once real time passed
2026-06-19 the skip slid the cutoff forward and aged the seeds out → `failed == 0`.
The bot's `run` job was `needs: test`, so every drafting-mode scheduled run failed at
pytest and the bot was skipped. It was *masked* because the `:30` `auto_publish_due`
cron (which skips the test job) kept passing, so the failures auto-closed and looked
like transient blips.

- **[#323](https://github.com/andrewzp/theheat/pull/323) — de-time-bomb the test** (`a9a4905`). Anchored the seeds to *now* via `_recent_health_timestamps()`; the three sibling tests no longer carry hardcoded dates. Verified 2028 passed.
- **[#326](https://github.com/andrewzp/theheat/pull/326) — decouple drafting from the full test gate** (`4b193e7`). The class fix: `bot.yml` `test` job now runs on `pull_request` ONLY; scheduled drafting runs skip it and pass a fast deterministic **Smoke gate** (`ruff check src/` + `python -c "import src.main"`). A flaky unit test can no longer dark — or redden — the bot. Drafts are manual-gated, so the residual risk is a rejected draft, not a bad tweet.
- **[#325](https://github.com/andrewzp/theheat/pull/325) — pytest ignores iCloud dups** (`db19c74`). `--ignore-glob='* [0-9].py'` so pytest never collects macOS/iCloud "name 2.py" sync-conflict duplicates locally (one carried the stale pre-#323 test). They were already gitignored, so CI never saw them — purely a local-DX guard. (5 such dup source/test files were also removed from the working tree this session.)
- **[#328](https://github.com/andrewzp/theheat/pull/328) — skip CI for the rolling daily-plan PR** (`699f6be`). [#207](https://github.com/andrewzp/theheat/pull/207) (`daily-plan-current`) is a content-only DISPLAY PR that never merges; its routine appends without rebasing onto `main`, so it re-ran the stale pre-#323 test and emailed a CI failure every day. `test` job now also guards `github.head_ref != 'daily-plan-current'`, so both jobs skip on that PR → neutral run → no more daily failure emails.

- **[#330](https://github.com/andrewzp/theheat/pull/330) — credential-expiry counters** (`de59f87`, MERGED). So a silent token expiry never recurs. `src/credentials.py` decodes the `exp` claim from JWT credentials each run (no network, no signature check; only the derived DATE goes to state — the token never leaves the bot) and `cli.py` writes `credential_expiry` to state. New state key wired through `DEFAULT_STATE` + `MERGE_SPEC` + `BotState` (`CredentialExpiry` TypedDict). Dashboard gets a **CREDENTIALS card** beside LAST HOT 10 (`lib/credential-status.js`, pure + tested; green >14d / amber ≤14d / red ≤3d-or-expired; exact date on hover). **Adding the next credential is one row in `TRACKED_CREDENTIALS`.** Verified: dashboard 95/95 `node --test` + clean `next build`; python ruff + mypy + 164 pytest. **Not yet live** — needs the manual `vercel --prod` deploy, and the card populates once the bot runs once with this code (next cron writes `credential_expiry`).

### Claim & warrant model — the design thread (DELIVERABLE, awaiting review)
The session began on this. A 2026-06-16 incident shipped precip drafts citing a
hardcoded 300 mm detection threshold (and a 0.0 mm baseline) as "the previous
record." Root cause is architectural: no typed model of a *claim* or its *warrant*,
so illegal states (a record with no baseline) are representable across every source.
- **[#322](https://github.com/andrewzp/theheat/pull/322) — design brief + integrity-gap notes** (`d99b317`, MERGED). The verified data-flow trace, five-whys, and reframe.
- **[#324](https://github.com/andrewzp/theheat/pull/324) — reviewed design doc + implementation plan** (OPEN, docs-only, **awaiting Andrew**). Reframes a claim as *the reference it cannot exist without*: 6 types (Record↔Baseline, ThresholdExceedance↔Threshold, Anomaly↔Climatology, Milestone↔MonotonicSeries, Categorical↔Authority, Magnitude↔nothing). Maps all 32 event dataclasses onto it; additive dark-launched `claim` field; **subsumes BOTH** honesty gates (Phase D cross-signal AND §F regional_anomaly). Both cross-reviews ran (codex `exec -s read-only` ×2 + `/plan-eng-review`) and returned REVISE; they **converged on the same P0** — the writer sees the whole bundle incl. `raw_signal_dump=asdict(event)` (62 callsites), so the fix is a writer-projection / verifier-evidence split (now §5f). All folded in. Docs: `docs/plans/2026-06-22-claim-warrant-model-design.md` + `-implementation-plan.md`.
