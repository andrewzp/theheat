## What shipped

### #345 — world-half eval-gating fix (`fix/world-half-eval-gating`, merged `9922169`)
The PENDING defect from 2026-06-27: `_run_world_cached_half` warmed (archive endpoint)
before evaluating (forecast endpoint) under one shared `saturated` flag, so the archive
429 tripped the flag the eval loop was gated on → the world half evaluated **zero** cached
cities for records every bootstrap run (`cov:0.0`). Reworked into three phases under one
cache read/write: **EVAL** (forecast; collect only) → **WARM** (archive; refresh stale) →
**CONSOLIDATE** (re-evaluate every pending city vs the freshest thresholds, finalize
`om_bundles`/country records, stamp provisional). Saturation split into
`eval_saturated`/`warm_saturated` (eval gated only on its own); `classify_world_status`
rewritten (warm saturation tolerated only during healthy bootstrap growth; steady-state
keyed to **pre-run** entry count); `prev_cached` now the true pre-run entry count, not the
lagging `_meta.cached_count`. Refute-before-emit + freshness-honest `as_of` preserved.
codex round 1 REJECT caught two real in-scope defects (`prev_cached`/`_meta`, falsy-`as_of`
false-freshness) → fixed → round 2 APPROVE. +20 tests
([tests/test_world_eval_gating.py](/Users/andrewpuschel/Documents/Claude/theheat/tests/test_world_eval_gating.py)).

### #347 — reganom recency window + ended-spell honesty (`fix/reganom-recency-window`, merged `0c710d1`)
**Why:** the France/UK heatwave was real and huge (live ERA5 dry-run: France +12–14°C for
7 days Jun 21–27; UK +7–13°C) but reganom only fired if the spell still qualified on the
**most-recent** complete ERA5 day. Both eased on Jun 28, so the trailing-run logic returned
`None` — a week-long +13°C event produced no draft. We activated mid-spell last session but
the first drafts were fact-check-killed (pre-#343) + onset-suppressed; by the time the guard
was cleared the spell had eased by a day. We missed it by ~24h.

**Fix** ([src/data/reanalysis_anomaly.py](/Users/andrewpuschel/Documents/Claude/theheat/src/data/reanalysis_anomaly.py)):
`RECENCY_DAYS = 2` — `detect_regional_anomaly` enumerates all maximal qualifying runs and
fires on the **most-recent** sustained (≥`min_days`) run whose last day is within 2 days of
the latest complete day. The magnitude bar (**+6°C / ≥2σ / ≥50% support**) is **unchanged**
— only recency widened. Robust against an isolated recent spike masking a genuine
just-ended spell.

**Ended-spell honesty (codex P2, 3 review rounds to converge):** an ended spell must never
read as current. The event now carries `latest_complete_day`; the bundle
([src/two_bot/intern/temperature.py](/Users/andrewpuschel/Documents/Claude/theheat/src/two_bot/intern/temperature.py))
computes `ended_days_ago` and exposes window dates; the writer prompt
([src/two_bot/prompts/writer_prompt.py](/Users/andrewpuschel/Documents/Claude/theheat/src/two_bot/prompts/writer_prompt.py))
requires **past tense + the window** when `ended_days_ago > 0`; the fact-check prompt
([src/two_bot/prompts/fact_check_prompt.py](/Users/andrewpuschel/Documents/Claude/theheat/src/two_bot/prompts/fact_check_prompt.py))
adds rule **j2** (an LLM recency gate that judges *meaning*, catching "bakes/grips/roasts"
that substring lists can't); and when ended, the bundle appends present-tense phrasings to
`forbidden_claims` for the deterministic §F gate. codex round 3: clean APPROVE, zero findings.

**Result — the France draft is in the `manual_only` queue** (pending), 262/280 chars:
> Across 6 sampled cities in France, temperatures **ran** 11.53°C (20.8°F) above the
> 1991–2020 daily normal **for 8 straight days through June 27** — 2.8 standard deviations
> out. At that margin, even brief nighttime relief disappears; the heat debt compounds hour
> by hour.

**UK & Ireland did NOT draft** — the writer produced >280-char tweets on all 3 attempts
(289–290; the longer "United Kingdom & Ireland" label + the fact template overflow 280), so
it was killed for length and is now **onset-suppressed** (`reganom_last_fired` has
`United_Kingdom_&_Ireland: 2026-06-21`). Surfacing UK needs a small writer-length fix + a
guard clear. Low urgency — France carries the European-heat story.

## ⭐ NEXT SESSION — P1: fix the boring/under-selling reganom tweets

Andrew's exact verdict on the France draft: **"this really is underselling what has been
happening."** It's honest and accurate but reads like a **stats bulletin**, not the story.
The writer is **Claude Sonnet 4.6** (`THEHEAT_WRITER_MODEL` default in
[src/config.py](/Users/andrewpuschel/Documents/Claude/theheat/src/config.py)) — plenty of
taste; the flatness is the **reganom writer-prompt section** steering it to open with
methodology + lead with the raw number + cite the z-score as a beat.

**What undersells it:** (1) methodology-forward opener ("Across 6 sampled cities in
France…"); (2) false precision ("11.53°C (20.8°F)"); (3) jargon as a beat ("2.8 standard
deviations out"); (4) it never says *how extreme* this is — a +11°C daily anomaly sustained
8 days means France effectively ran a different, hotter climate for over a week (the
"wait, what?").

**The task (Andrew's words, verbatim):**
> "upgrade the reganom writer-prompt section to lead with significance, weave the
> sampled-cities attribution instead of opening on it, round the number, use σ as awe not a
> beat, and name the stakes — strengthening the editorial bar, not weakening honesty (it
> stays fully inside the claim/warrant gates, then runs through fact-check + critic + a
> codex pass)."

Illustrative target voice (same honest facts — sampled cities, past tense, no national
overclaim — but it lands):
> Six French cities just ran ~11°C above their 1991–2020 normal for **eight straight days**
> — a sustained 2.8σ anomaly, the margin where "above average" stops describing it. For a
> week, France was effectively living a hotter country's summer. It eased June 27.

**Scope/where:** edit the "Regional anomaly bundles" section of
[src/two_bot/prompts/writer_prompt.py](/Users/andrewpuschel/Documents/Claude/theheat/src/two_bot/prompts/writer_prompt.py)
(the framing rule that currently says cite "6 sampled cities in [Region]" *verbatim as the
opener* — change to "attribute to the sampled cities, but lead with significance"). Keep
ALL honesty constraints intact: sampled-cities attribution (never bare-region/national
average — the §F `forbidden_claims` + fact-check §j gates stay), past tense + window when
`ended_days_ago > 0` (§j2). Consult the voice memories: editorial-bar "wait, what?" test;
the Attenborough/Economist "place the data point inside the system" voice; the banned
voice-failure patterns (press-release openers, label:value, false precision). Then:
**regenerate the France draft** (clear France from `reganom_last_fired` via the gist reset
if needed + dispatch an `alerts` run, OR run the writer in a dry-run harness) and confirm
the new draft lands harder while passing fact-check + critic. Run a **codex-xhigh pass** on
the prompt change. **Then make it great** — iterate the voice until Andrew is happy.

**Down the road (DEFERRED, not now):** weave real **news anecdotes** into tweets ("buses
crashed because the drivers passed out"). The hard constraint that defines that feature:
current news is *outside the writer model's knowledge cutoff*, so anecdotes must come from
**real, sourced retrieval** (grounded search), never the model's imagination — otherwise
it's hallucination, the one thing that breaks the no-false-claims bar. It needs its own
brainstorm → design → build. We started the brainstorm this session and paused.
