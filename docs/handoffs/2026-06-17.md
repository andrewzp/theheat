## What shipped this session

- **PR #304 — `voice-regression` permanent fix** (`f991029`). The daily live
  writer-replay had failed every night **2026-06-12 → 06-16** (last green 06-11).
  Two independent causes, neither a voice regression:
  1. **`usgs_earthquake_bundle`** — @theheat is a *climate-data* account; a bare
     M7.1 quake has no climate mechanism, so the live model kept (correctly)
     declining it — but *non-deterministically*, so the test flickered. The test
     demanded a tweet every run.
  2. **`regional_anomaly_bundle`** — its honest "N sampled cities" framing runs
     ~286 chars, so the writer correctly kills rather than ship >280. ~50/50 by
     sampling.

  The architectural flaw: the suite treated **any writer KILL as a regression**,
  but the writer's own rule is *"killing is the default; a mediocre tweet is
  worse than silence."* A daily, paid, non-deterministic gate that hard-asserts
  "the model always emits a tweet" flakes forever.

  **Fix (two parts):**
  - **Deterministic editorial-scope guard** — `OUT_OF_SCOPE_SIGNAL_KINDS =
    {"usgs_earthquake"}` in [`src/two_bot/writer.py`](/Users/andrewpuschel/Documents/Claude/theheat/src/two_bot/writer.py).
    `write_tweet` kills out-of-scope signals **before any model call** — free,
    reliable, unit-tested in the *regular* suite
    ([`tests/two_bot/test_writer_scope.py`](/Users/andrewpuschel/Documents/Claude/theheat/tests/two_bot/test_writer_scope.py)).
    The USGS source still runs (it backs GDACS subtype-witness redundancy); only
    the climate-voice *drafting* of bare quakes is declined.
  - **Voice-replay suite realigned to the writer's contract**
    ([`tests/voice_regression/test_writer_replay.py`](/Users/andrewpuschel/Documents/Claude/theheat/tests/voice_regression/test_writer_replay.py)):
    a clean kill is acceptable for any bundle; the suite fails **only** when the
    writer SHIPS bad copy (>280, banned patterns, fabricated context, dishonest
    framing) and asserts out-of-scope signals are declined.

  **Verified:** dispatched the real workflow against `main` post-merge →
  **`40 passed in 7m13s`** with the live Anthropic writer. Gates: ruff clean,
  mypy clean (113 files), full regular suite 1998 passed.
