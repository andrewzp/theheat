### World threshold cache — #338 (`6fbd3a7`)
The architectural root cause was an asymmetry: the GHCN/US half is backed by a
precomputed threshold cache (cheap per-run compare); the world half recomputed 30 years
of history live every run. #338 gives the world half the same model:
- **Warm path** (`_run_world_cached_half`, `src/orchestrator/sources/open_meteo.py`):
  each run refreshes the ~8 stalest/missing cities' 30-yr archives (urgent-first), paced
  under an Open-Meteo weight budget; thresholds stored in a **separate gist file**
  (`world_threshold_cache.json`) — `state.json` untouched.
- **Hot path**: cheap multi-location forecast for cached cities vs cached thresholds —
  no archive call.
- **Budget + 429-resilience** (`src/data/openmeteo_budget.py`): a process-level
  `OpenMeteoBudget` paces the run under 600/min (with leaderboard headroom reserve); an
  Open-Meteo 429 now raises `OpenMeteoSaturated` and stops the run with **degraded**
  status — the original *silent* saturation is structurally impossible.
- **Field/month-wise cache merge** (`src/orchestrator/world_cache.py`): newest `as_of`
  wins per city; equal `as_of` keeps the more-extreme value per field — concurrent
  same-day writes never erase each other.
- **Anti-re-fire**: a fired record writes a provisional threshold so the same
  "hottest ever" can't re-tweet daily for 30 days.
- **Country-record coverage floor**: a national record only fires when enough of that
  country's cities are cached (no false "Spain record" from 1 city during bootstrap).
- **Failure classifier** (`classify_world_status`): steady-state degrades on low
  coverage / forecast-failure spike / saturation; bootstrap degrades only if the cache
  **stalls** run-over-run.
- **Scope**: `calendar_date` / `record_streak` / `simultaneous_records` are now
  **US-only** for the world (gist-prohibitive to cache; weakest tiers).
- Modules: `src/data/world_thresholds.py` (`CityThresholds`, `compute_city_thresholds`,
  `evaluate_city`), `src/orchestrator/world_cache.py`, `src/data/openmeteo_budget.py`.
- Spec: `docs/superpowers/specs/2026-06-26-world-threshold-cache-design.md`; plan:
  `docs/superpowers/plans/2026-06-26-world-threshold-cache.md` (both 2× codex-reviewed).
