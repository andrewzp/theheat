## Adversarial review (codex + 12-agent Opus panel)

Both converged on the same findings; all fixed before merge (commit `0f5676d`):
false-recovery auto-close on transient fetch failures (now closes only on
confirmed recovery; bails on total outage; `None`-not-`{}` on unlistable issues);
the meta-guard couldn't read the beacon with the default token (now env-first via
`vars.SELFHEAL_BEACON`); dashboard masking via `per_page=1` (now picks latest
*decisive* run); runbook autonomy let test-contract changes auto-merge (now
Judgment/PR-and-stop) + cross-run anti-thrash + sentinel self-liveness step.

## ⚠️ The one open step (Andrew)

```bash
gh secret set SELFHEAL_PAT   # PAT: repo scope (+ Variables: write if fine-grained)
```

Then watch the first run:
```bash
gh workflow run workflow-self-heal.yml --ref main
gh run watch $(gh run list --workflow workflow-self-heal.yml -L1 --json databaseId -q '.[0].databaseId')
```

Why required: a PR opened by the default `GITHUB_TOKEN` does **not** trigger the
`test` gate, so the routine couldn't merge its own fixes on green; and the default
token can't write the beacon variable. Without the PAT the routine still runs
**detection-only** (Layers 1+2 give the loud signal regardless).
